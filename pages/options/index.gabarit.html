? Activer tous les flux

Sur les anciens versions du module, une erreur HTTP conduisait l'état du flux sur "inactif". Plus aucun article n'était donc récupéré pour ce flux. désormais, seuls les derniers états HTTP sont gardés.

rendre actifs tous vos flux?

? Etat de la base de données

Attention : ces opérations touchent sans distinction toute la base ; y compris ceux qui sont épinglés ou mis en favoris. Gardez bien à l'esprit que le fait de vider la base des articles par exemple, sans supprimer les flux, pourrait avoir des conséquences inattendues : les extractions continuant, la base se remplira avec de nouveaux articles.

type(s) de données à supprimer
confirmation

? Relancer le module

Parfois et malgré le soin apporter à la programmation, il peut arriver que des petits bugs apparaissent. La relance manuelle - qui prend effet immédiatement -, permet parfois de corriger ces défauts temporaires.

Une fois le module relancé, il est conseillé de redémarrer le navigateur.

Ne tentez pas de relancer le module si cela n'est pas nécessaire : l'arrêt de la connexion du module à sa base peut être brutal et provoquer des pertes de données indésirables.

confirmez-vous arrêter les connexions et relancer du module ?

? Accès au débug

Cela vous donnera l'ensemble des retours du module - ce qui représente une très IMPORTANTE quantité de données à afficher. N'activez cette fonction qu'avec parcimonie et plutôt sur des périphériques fixes.

la fonction fait apparaître les outils sur toutes les pages une fois actif
